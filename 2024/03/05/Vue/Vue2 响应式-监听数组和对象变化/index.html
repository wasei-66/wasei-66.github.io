Vue2 响应式-监听数组和对象变化

发表于2024-03-05|更新于2024-03-05|Vue
|阅读量:

一、响应式原理

利用 ES6 的 Obeject.defineProperty() 来监听属性变动，将 data 里的数据对象进行拦截，递归遍历，包括子属性对象的属性，都加上 setter 和 getter，若给这个对象的某个值赋值，就会触发 setter，以达到双向绑定的效果

所以 vue 会监听 data 中所有层级的数据，需要响应式的数据要在 data 中进行初始化定义

由于 JavaScript 的限制，Vue 不能检测数组和对象的变化，但是有办法来回避这些限制并保证它们的响应性

二、数组监测

1）可监听数据变化场景

  • 通过赋值的形式改变正在被监听的数组
  • 通过 splice(index，num，val)、push 等重写方法改变正在被监听的数组

vue2 对 pop、push、shift、unshift、splice、sort、reverse 这些方法进行了重写，调用这些方法是可以监听到数组的变化的

vue 包装的数组方法做了什么 ?

  • 1.正常调用原生数组方法

  • 2.重新解析模板，进而更新页面

2）无法监听数组变化场景

  • 直接通过数组下标进行赋值 arr[i] = value
  • 直接修改数组的长度 arr.length = newLen

其实 Object.defineProperty 是有监控数组下标变化的能力，因为大多数情况下对数组的操作就是遍历，如果用 Object.defineProperty 监测它的变化，就会有很大的性能问题，正如尤大的“性能代价和用户体验收益不成正比，所以出于性能/体验的性价比 vue2.x 并没有实现这个功能

3）解决方式

修改值

  • this.$set(arr，index，newValue)
  • Vue.set(arr，index，newValue)
  • arr.splice(index, 1, newValue) 先删除后添加
  • 使用临时变量作为中转，重新赋值数组

修改数组长度

  • arr.splice(newLen) 截取数组

二、对象监测

1）可监听对象变化场景

  • 直接重新赋值

2）无法监听对象变化场景

  • 对象属性的新增和删除

3）解决方式

新增属性

  • this.$set(obj, newKey, newValue)
  • Vue.set(obj, newKey, newValue)
  • obj = Object.assign({}, obj, {newkey1: newvalue1, newkey2: newvalue2}) 合并对象

删除属性

  • this.$delete(obj, key)
  • Vue.delete(obj, key)
